[ 3 Learn VueJs In Arabic - What Is Single Page Applications] Concept:

=> [SPA] [Single Page Application] Concept :
  • An web Application of an only single HTML page
  • This Application is updating this page with any User interactions of this page Dynamically
  • At Normal (Specially with large Web sites and Web Applications) There are Multiple pages ,
    But with Vue.js There is another Method to handle this => {Rout}

=> [MVC] [Model View Controller] Concept :
  • An original Software Architecher , which is seperating between both interface[View] and Business [Model] with a between Connection [Controller]
  • Life Cycle of the [MVC] as folowing:
    1. [View] -> Which is the user interface, where user is interacting with the page such as
      • Clicking
      • Navigating
      • Applying a Process such as:
        1. Adding/Deleting a product to a product cart
        2. Adding/Deleting a friend to a Friend list
        3. Open/Deleting a message from a message Box

      [View] Function is => Any interaction will be sent to the [Controller]

    2. [Contoller] => Which the Connecting chain between the [View] and [Model], and it's function as foolwoing :
      1. Recieving the User's inputs from the [View]
      2. Sending the User's inputs from the [Controller] to the [Model]
    3. [Model] => The main Application data storage (like the Database)

=> [MVVM] [Model View ViewModel Model] Concept :
  • The [Vue.js] Software Architecher , which is seperating between both interface[View] and Business [ViewModel] with a between Connection [Model]
  • Much Better than the original Architecher
  • Dealing with View and Model by twice Asynchronized processing
  • Life Cycle of the [MVVM] as folowing:
    1. [View] => Which is the user interface, where user is interacting with the page , just as the original [MVC] architure
    2. [ViewModel] => Which is Dealing with the View and Model processings at the same Time , with Tripple Actions , as followig :
      1. Recieving the inputs from View of users
      2. Connecting to the [Model]
      3. Updating the View Interface Directly
