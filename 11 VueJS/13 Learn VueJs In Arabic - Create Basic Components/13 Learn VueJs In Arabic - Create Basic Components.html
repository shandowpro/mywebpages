[ 13 Learn VueJs In Arabic - Create Basic Components] Concept:

=> [Reusable Components] Concept :
  • [Reusable Components] =>
    • One of the Vue Model Parts, but has it's own Sub model in the Vue Script
    • One of the most powerful concepts an tool in any Framework and various Libraries
    • Used for the repeated Websites Tempaltes Applications components
    • {Reusable component} => Syntax is similer to the {on()} function in {JQUERY} => [ Vue.component('component-name', {data: function(){return {propName1 :"propvalue1", propName2 :"propvalue2", } }, template: 'element syntax html & vue' } ) ]
    • {Reusable Components} => Has it own Sub Model for each Component with in the main same Vue Model , and the (2) Parts :
      1. {Data} => with following aspects :
        1. Contains the new component Data that will be pass through into template Elements like [Properties && Methods]
        2. It is about a function of returning object elements of passed data
        3. Contains the new component Data that will be pass through into template Elements like [Properties && Methods]
      2. {Template} => Contains the Template HTML Elements and Components like : (<p>) (<div>) (<h1>)
    • {Reusable Components} Best Practies => :
      1. {Repeated Elements} -> VueJS {Reusable Models}
      2. {Unique Elements} -> VueJs Arrangements

  • The Differences between properties passed through the vue inxtances and properites passed through the vue compnonents as following :
    • [Vue Instances properties] =>

      will be public passed through any element in the {vue application} , not just in the {vue new component}

      Declaration method -> can be declared in the [Vue instances model] properties [with in (Data:) section ]

      calling method -> can be Called by inside vue { { } } , in any place of the Vue application [[but Can Not be Called in new declared Vue component]]


    • [Vue new component properties] =>

      will be private passed through only just in the {vue new component}

      Decleration method -> can be declared in the [Vue new component model] passed properties [with in (Data:) section ]

      calling method -> Called by inside vue { { } } , but only to be called and used in the new Vue component

    Niether {Defined Properties of Vue Instances} can be used in the {Vue new component}
    NOR New {Defined Properties of Vue new component} can be used in the {Vue Application outside the defined Vue component}

  • The full Ideal aned correct priorities in the Vue model until now:
    1. Vue Application
    2. Vue importing library script -> [calling Vue file]
    3. Vue Model => including the following (in order) : (3 parts)
      1. Decleration of the [new component] -> including following : (2 parts)
        • [Vue new component data] => the data(properties) will be passed through the new component
        • [Vue new component Template] => the Syntax of the new component
      2. Decleration of the [Vue Intances] -> will be passed through the Vue Application -> including : (2 Parts)
        • [Vue application Properties]
        • [Vue application Methods/functions]
