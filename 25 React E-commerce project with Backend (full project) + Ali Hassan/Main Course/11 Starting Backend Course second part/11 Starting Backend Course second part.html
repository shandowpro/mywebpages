Main Major steps of the our project [E-Commerce ] :
  1. Creating the React app using the {Vite} -> [done]
  2. Designig the React app using the {MUI: material user interface} -> [done]
  3. Downloading the images of [Hero section , (6) products] of the React from the {courses4arabs.org} website -> [done]

  5. Creating Backend [CMS: content management system ] using the {Strapi} -> [now doing]
  6. Deploying the project website files using the {Render.com} server -> [later]

Main concept of the {Strapi} CMS :
  • It is a headless open source code Content management system used to manage , operate and deploy the backend of the project backend server and database
  • [headless] => means it deals with any framework used to create the UI design
  • This means that it provide [API] and [Database] for your backend server side
  • The provided [API] => is in a form of array of objects , which can called and used inside the project UI , as following :
    1. [API] => it is about a a URL json extension generated and obtained from the sever [CMS] provider , which is a type of json file
    2. [API] => it can be read from the and can be read on the internet browser [via spceified json reader browser addon] , and the code editor [vsCode]
    3. Each array => represent a collection of data stored with in the Strapi Conntents management system
    4. Each object [a member of the main API array] => represent a a single item of stored in this database [Data Cellection] , and it consists of several properties [several data of each row
    5. To create and mangage content of data => we need to instal and setup the {strapi} sever and libarraies files within our project by using certain steps using [Node Js]
    6. It is reconmmended to add a plugin on our web brouser [such as : 'Json reader' ] => to make it easier to read the obtained Json files of the [API]


Main Features and benefits of the {Strapi} CMS and it's dashboard:
  • Provide a [Database] to store/retreive and deal with project's data
  • Provide a [API] to obtain data from the Strapi sever via using it's url in the UI codes
  • Provide a Content managaemnt system as modern dashboard of controling tool of project data and content
  • Provide a Content managaemnt system as modern [Dashboard] of controling tool of project data and content
  • Strapi's [Dashboard] 's properties => :
    1. Can be used for both :
      1. Basic data Structure {schema} and contents [such as : product's data] => will use it first in the course
      2. Users data Structure {schema} and contents [such as : user's data] => will be used later in this course
    2. Provides Control Panel to control and deal with all Data Collections , by :
      • Advanced Displaying screen with filteration abitlites of exsited [Data Collections]
      • Ability to [add , modify , delete] any of added item(s) [row/entry of data]
  • [Data Structure] for each single [Data Collection] type , has the next features :
    1. Ability to add/Modify/delete the following of data types [data fields] :
      • Textual Data [long , short] => [Title / Sub Title / Description]
      • Numeric Data [decimal , Integar , long] => [Price / Rating]
      • MultiSelction Menu [Categories , Sub Categories]
      • Media [Images , Videos , Sound]
    2. Ability to determine the {Obligatory status : Required to be added || Not Required] for each data field's}
  • Each Row/Entry added in a single Data Collection , has the next features :
    1. Ability to add [single/multiple] image(S) for a single entry/product + assinging priority of displaying
    2. Ability to choose among available [Categories - added within the data structure ] for a single entry/product
    3. Ability to [save] and/or [publish] each added entry to teh assigned data collection



Main Requirments for installing and setup {Strapi} server in our project :
  • A] Preperation procedures and steps :
    • 1- Project files arrangment : by seperating the projects files into (2) main folders :
    • [ fontend] folder => will contian the UI Desinging files [the previous React created files ]
    • [ backend ] folder => will contain the [strapi] files (which will be created in this lesson and later) including :
      1. CMS system
      2. Server fies
      3. database shcema
      4. RTK Query liabrary
    • 2- Downloading and install the [NVM] library :

      This is an overriding runtime of [node.js] to provide a conrtrol [install, use, overriding ] system on all nodesjs available versions, and choose the applicalable versions [LTS versions : v14 . v16 . v18] with the [Strapi]

      Installing and setup this version using the next steps :

      1. Install the downloaded [NVM] source on pc , then restart the pc
      2. open the project [backend] folder => [cmd prompt] on the vs code
      3. use the following commands - in order :
        • Display the current list of installed Nodes JS versions , using the next code :
          nvm ls

        • Install the desired LTS version of nodejs that matching with the {Strapi} => {18.16.0} :
          nvm install 18.16.0

        • Assign the desired installed LTS version of nodejs that matching with the {Strapi} to bve used as overriding =>
          nvm use 18.16.0


    • 3- Downloading and install the [Strapi] application :

      After installing the compatable version of the Node, we will install the {strapi } application it self in the current directory of [backend] , by using the next code + it's entries :
      npx create-stapi-app@latest
      project name => [ ./ ] to save the strapi projectin current folder [backend]
      Installation type => [ | ] to save the strapi projectin current folder [backend]
      [Quick start] => we select it [for now as we are starter]
      [Custom (manual settings) ] => we will select it later [ when we bet ready for uploading the project files ]

  • B] Run the installed application [strapi] server :

    After the Strapi files has been installed and setup successfully on our prjoect files [inside the backend dirctory] , we gonna run/operate the [strapi] server up by using the following code :
    npm run develop => [this code will run the strapi server , and generate a strapi sever port to open the account of the strapi , and open it automatically ]
    npm run dev => [this code will run the created UI react design of the project , and generate a react server port to open the account of the strapi , and open it by click on it ]

  • C] Open the generated link of [strapi] server :

    After running the Strapi server, we gonna open the generated link of [strapi] server , in order to accesss the Registration Strapi UI , to create a new account in the Strapi , using the email

    After creating a new account on the {Strapi} server => we will get access automatically into to the Strapi Dash board ,to control our [CMS] of our project

  • D] Getting our free main API [Rest Api] Link from Strapi :

    To obtain the [API Link] is the main purpose of using [CMS] of Strapi of any other servers , which is e gate of controling the project data [get/obtain & send/store]

    There are (3) main steps To obtqain the [API Link] on the Strepi :

    1. 1- Build the {Content Structure} of our project data , which represent the [database] of our project , using next procedurss :
      • plan our project main data collection Name & main fields datatypes :

        Our project main [data collection] Name => {Product} : [must be in single form -Not plural - ]

        Our project main {Product} [fields datatypes] =>

        1. [Product Title ] => short text / required
        2. [Product Description ] => long text / required
        3. [Product Price ] => Decimal number / required
        4. [Product Image ] => Media / required (single OR multiple)
        5. [Product Rating ] => Decimal number / required
        6. [Product Category ] => Enumeration / required ( men OR women )
        7. [Save]


    2. 2- Creating/Adding data Entries of each row [product] , an following steps :
      1. [Content Manager] => select {Product} => [+ create new entry ]
      2. filling all data of product [as assinged data structure] => and then [save] & [publish]
      3. After [publish] the saved entry will be saved and published on the database [data collection]
      4. After [save] all {Content Structure} of created data collection & publising the first entry of data => The API link is able to be obtained , by the next step
    3. 3- Getting the generated API link from :

      {settings -> Edit role => Users & Permissions Plugin => Public -> Permissions -> Product -> [find ] : this will generate the [Api url link] }

      The generated [Api url link] => ' localhost:1337 / api/products ' -> by using this url => you will get access to all your data of [Product]

      Our generated [Api url link] => can be accessed after logging in to our [Strapi] account server

  • E] Getting our free Image API Link [Rest Full Api] from Strapi :

    The previous link web obained is NOT including the Media contents [such as images] , which as the next pattern :
    localhost:1337 / api/products


    To obtain Full link that including the Media contents [such as images] is called {Rest Full Api} , and to obtain this full link by using the next pattern :
    localhost:1337 / api/ products ? populate = *


    To obtain the high resolution of the image url [beside another image url of low iamge resoluion] , form the inner path inside [Rest Full Api] link [in form of json structure ] :
    {
    "data" : [
    {
    "attributes" : {
    "productImg" : {
    "data" : [
    {
    "attributes" : {
    "url" : "/upload/final_..... .png " => [this value is the product image url of this product ]
    attribute prop name " : "prop value "
    attribute prop name " : "prop value "
    }
    }
    ]
    }
    }
    }
    ]
    }

  • Default pattern of the API link :

    The returnd api link is about json object with name ["data"] , which is about an [array of objects] , and each object is reperesent a single element , and it can ber as jnest form :
    {
    "data" : [
    {
    "attributes" : {
    "productImg" : {
    "data" : [ { "attributes" } ]
    }
    }
    }
    ]
