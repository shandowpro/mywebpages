13 Free Deployment of backend strapi on render com:

General differences in concepts between { Frontend files deployment service provider } & { Backend files deployment service provider} in this project :

in this project [fulstack] we will use both of frontend & backend deployment , as following :

  1. [render.com] & [cloundinary.com] => Free deployment services for [Strapi projects backend files & iamges]
  2. [firebase.com] => Free Hosting services for [Strapi (and other types) projects frontend ]


General differences in concepts between { free Strapi deployment backend files service provider [render.com] } & { free Strapi deployment images service provider [cloudinary.com] } :

Both of [render.com] & [cloudinary.com] are specifed Free deployment files services for [Strapi projects] , with folowing differneces :

  1. [render.com] => Free deployment services for [Strapi projects backend files]
  2. [cloundinary.com] => Free deployment services for [Strapi projects imags files]


Other paid services for Strapi deployment backend files :

There are other sevices providers than the [render.com] , but with paid fee , such as :

  1. AWS Deployment service provider
  2. Azure Deployment service provider
  3. Digital ocean app platorm
  4. Digital ocean droplet Deployment
  5. Heroku

  8. Username `=> RENDER username
  9. Password => RENDER Password

Main practical steps of deploying the backend project files on the free deployment service provider [render.com] :
  1. Before install strapi => we need to copy of the previous project of the [E-commerce ] (keep the {frontend} + Re-Create the [backend] folder , using the {render.com} the free database projvider service that dealing with [Stapi] )
  2. Start over creating the [E-commerce] using React {Vite} , with manual settings , by following steps :
    1. 1- Start the {Boiling plate} of [React-Vite] inside the {backend} folder of the project , and keep it awaiting

    2. 2- Creating new account at [render.com] , and start creating database procedures , by next steps :
      1. from the official websitge of the render.com => [get started]
      2. Creating a new account on the {Strapi} by using email
      3. Login on the new created accoount
      4. Access to the [dashboard] , and choose the [postgre] database type => new {postgre SQL} OR [new +] -> [postgre SQL]
      5. Assign the new database settings within in the starpi database , as following :
        • Name : ecommerce
        • Database : Ecommercedb
        • user : Shadipro
      6. Click on the [create Database] button
      7. While creating => status will display as [creating ...]
      8. After creating => Database Connections tap will be acceesible and available with the following information :
        1. [database] => will provide the [new database name] => (assingedNamedb_....)
        2. [Extended DatabaseURL] => will provide the [Database Host value] -> [take only form the 'dpg-' to 'render.com' ] (the database host not from the 'hostname' value )
        3. [Database port ] => will provide the prot number of Strapi databse deplyoment sevice provide
        4. [port] => will provide the port number of Strapi database deplyoment service provider

    3. 3- Continue filling the awaited {Boiling plate} of [React-Vite] ( form strapi connection information) to finalize Creating/setting up the [strapi app] by using the shorten cmd code {according to the Strapi database account } :
      npx create-strapi-app@latest => and filling the next setttings :
      project name : backend => [to create backend Strapi files inside this new directory ]
      Installation type : custom => [ Manual settings ]
      Prefffered lanauage : Javascript
      Default database client : Postgres
      Database Name : [ Get it from the {Extended DatabaseURL - from the strapi Connections -> database account } ]
      Host : [ Get it from the {Extended DatabaseURL - from the strapi Connections -> database account } ]
      Port : [ Get it from the {Port number - from the strapi Connections -> database account } ]
      username : [ Get it from the { shadipro - from the strapi Connections -> database account } ]
      password : [ Get it from the { increpted password - from the strapi Connections -> database account } ]
      Enable SSl Conection : [ y ]

    4. 4- Setting up the [Cloudinary] images deployment service provider [related to th strapi] , by following the next steps :
      1. Getting the {Cloudinary} required docmentation for setting up and instaling wihtin my project : form the strapi -> dashboard -> marketplaces -> providers tab -> [Cloudinary] => more
      2. Get/copy the npm install cmd code , and write it inside the main folder of the project , as following :
        npm install@strapi/provider-upload-cloudinary--save

      3. Applying the steps of setting up project files to be with the available scripts to be compatable with {Cloudinary} iamages/media deplpyment service , as fololwing stpes :
        1. Adusting the [backend/config/plugins.js] according to the [Cloudinary] 's pattern {plugins.js}
        2. Modify the [backend/config/.env] , by add/define (3) variables , as next :
          // At the last of file [variables name as found in the {plugins.js} file ] & [varialbes values as found in the {cloudinary account website } ] :
          CLOUDINARY_NAME=value from the [cloudinary account website]
          CLOUDINARY_KEY=value from the [cloudinary account website]
          CLOUDINARY_SECRET= =value from the [cloudinary account website]

        3. Adusting the [backend/config/malewares.js] according to the [Cloudinary] 's pattern by => replacing the 'strapi::security' property name in {malewares.js } file by [object] found in the cloudinaray pattern

        4. Adusting the [backend/database.js] within the [connections] object variable -> [postgres] property -> connection => according to the [Cloudinary] 's pattern by folowing steps =>
          • Removing the , 'localhost' from -> host: env('DATABASE_HOST'),
          • Removing the , 'localhost' from -> port: env.int('DATABASE_PORT'),
          • Removing the , 'strapi' from -> database: env('DATABASE_NAME'),
          • Removing the , 'username value ' from -> user: env('DATABASE_USERNAME'),
          • Removing the , 'password value' from -> password: env('DATABASE_PASSWORD'),
          • Removing the , false from -> ssl: env.bool('DATABASE_SSL') && {

        5. Starting the backend project by {npm run develop} , to test the {cloudinary} responding ,as flowing :
          1. Checking out the [media library] of [cloudinary] before adding any images inside the [strapi] => should be empty
          2. Adding new entries inside the [strapi] including the images [save + publish]
          3. Get access to the [second role] from the settings -> public permissions -> content type [ product ] => find => get the [api url - with using full api url ]
          4. Get access the [api url] of [product] content type to test the accebitlity of this url json file
          5. Checking out the [media library] of [cloudinary] again after adding any images inside the [strapi] => should be found [with different sizes for each image ]

        6. Start testing the [frontend] responding {UI} to the [strapi] and [cloudinray] imags deployment provider service , by following steps :
          1. Start the [cmd] => {npm run dev} => to run the UI vite frontend
          2. Removing the (image prefix - which was including the [.env] defined localhost variable $ import.meta.env.VITE_BASE_URL - ) url used inside the images src inside the [CardMedia] -> 'image' property , [due to we are now using the {cloudinary} host database ]

    5. 5- Start uploading the full project [frontend & backend ] to the github repo [by default the uploading will not include the (.env -> which including classified data and information about the [strapi & Cloudinary ] database sever ) ]
