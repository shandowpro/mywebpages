19 React Axios API Requests Axios with React JS Tutorial concept :

A] Main concept and mechanism of using {axios} react library with curd operatsions :


  1. The [axios ] is about an [api] setting up for the react application , that creating a main connection api url [baseURL] with assinged {watheched} database json file by the json server , and that could be done by using the main baics configuraions steps :
    • a] Creating the main database json file inside the project's main root in a data folder 'data' => [src/data/db.json] , as an json object file

    • b] download the main [axios] library to be used in to define {api} & [Crud opertations] with real api connections , by using the next npm command :
      npm i axios


    • c] Creating and define the main [api] configuration file using installed and imported [axios] library -> [api/posts.js] to be used to define the main axios {baseURL} to be used as prefix [which will be assign by 'localhost' in this project , but will be replaced by a live api in the future ] to be used by json server to access the defined database from the assigned json file defined inside the [data] folder , an this file will include the next definition :
      // importing the main axios library to be used in define and create the api :
      import axios from "axios"

      // Exported creation function extracted from [axios] imported library ,that include the main base url prefix + port nubmer to be used in any connetion crud opertation with the api :
      export default axios.create({
      baseURL: 'http://localhost:3500' ;
      })


    • d] Activating the json server by using [npx] with assigning the each of both flags [port , watch ] :
      1. assign the used port number {localhost in this project } to be used to as json server , by using the next flag :
        -p 3500

      2. assign the database file to be watched from the assinged json server [this step need inital internet for the first time at least ] , by using the next flag :
        -w src/data/db.json
      3. Full command s next code :
        npx json-server -p 3500 -w serc/data/db.json


    • e] Modify the main {App.js} with next steps :
      1. 1] New importings :
        • a- import the [axios] library to be used to assing the main crud opertations => {import axios from 'axios'}
        • b- import the defined api by axios [api: as custom name] from the api definition file [to be used in any connection with api ] => { import api from './api/posts' }
        • c- importing the required classes from [date-fns] library

      2. 2] Modifications of Defined States :
        • a- keep the defined state of [posts, setPosts] , but with empty array of objects => [to be obtained from the api , instead of the static array ] , by the next code :
          const [posts, setPosts] = useState([]) ;

        • b- Defining required states for [add/creating a new post] [title] and [body] :
          const [postTitle, setPostTitle] = useState('');
          const [postBody, setPostBody] = useState('');

      3. 3] Modifications in Defined {useEffect} states callback functions :
        • (a) Modify {Fetching data} first CRUD operation => inside the main useEffect including [fetchPosts()] function => responsible for reading / fetching the returned data from the assinged [api] & [json data] file , but using the {axios} method instead of using the old method of {fetch() } , by using the next steps :
          1. a- modifications inside the try{} :
            • 1- Must use the {async - await } method within in the inner main funcion definition {fetchPosts()}
            • 2- Define the variable of the [response] with value of the imported [api] with using [.get()] method to get the data from the assinged api path ['/posts'] as sub api path after the baseURL of the defined [api] ,as the next code :
              const response = await api.get('/posts');

            • 3- Assigned the response variable value inside the [posts] status , by using directly the defined respnose's extracted [data] :
              setPosts(response.data);


          2. b- modifications inside the catch() {} :
            • 4- => inside the [catch{}] :
              catching and handling the default error message (err) that is being generated automatically by printintg the builtin error [if it is existed ] output that handled by axios [with using axios handling options ] :
              } catch (err) {
              // print the builtin error output that handled by axios [with using axios handling options + and defined variable 'response' ] after checking if the resposne error is existed and already gernareted :
              if (err.response) {
              console.log(err.response.data)
              console.log(err.response.status)
              console.log(err.response.header)
              } else {
              // print a custom error message incase there is no handled output displayed default [axios] :
              console.log(err.message)
              }
              }

          3. c- executing the upper defined function {fetchPosts()} :
            1. // executing the defined {fetchPosts} function inside the useEffect , by using next code :
              fetchPosts();

          4. d- adding empty array as dependencies of the useEffect [] => to be activated at the first time of running of the project -> to obtain the fetch for data by start the page

        • (b) Modify {Post/Add new item data} second CRUD operation => {handleSubmit()} function repsonsible for posting/adding new data item into the assigned database through the assigned api , usign the {axios} method instead of {fetch} using next steps :
          1. 1- before the try{} :
            • Adding {async} to the main function with using the (e) paremater to used to prevent the defualt submitting
            • keep the defined variables of {[id] property} & {[dateTime] property} & {newPost} as it is without any modifications
            • Define the {newPost} as object variable of full new post object [with assigning all object properties - in order - ]

          2. 2- wrapped inside try{} and catch{} :
            • a- Define a variable of [response] that as {api posting connection} , using the imnporetd deifned [api] with '.post()' method , with paramters of :
              • First parameter [posting path] => '/posts' => which is the file name defiend inside the 'api' folder

              • Second parameter [new object of item of data] => assinged by value of upper Define a variable {newPost} of full new post object [with assigning all of objtect properties as following :
                1. [id] : represent the upper defind [id] property
                2. [title] : represent the defeind state of [postTitle] as incomming prop
                3. [dateTime] : represent the upper defeind property of [dateTime]
                4. [body] : represent the defeind state of [postBody] as incomming prop


            • b- Define a variable of [allPosts] as array of summition of both (2) follwing :
              1. { ...posts } => the seperated of main incoming datasource 'posts ' from the assigned api
              2. { response.data } => the new upper defined { response } extracted data which is represent the new posted object 'after being assined '

            • c- Adding the new variable of all posts into the [posts] state using its setter its function by the upper defined variable of assembled array 'allPosts' -> { setPosts(allPosts )}
            • d- Setting the [postTitle] & [postBody] states into an empty values after adding the new post, using their setter function :
              1. setPostTitle('')
              2. setPostBody('')
            • e- directing back to the home page after implementing the posting operation , by using the defiend history with path of the main home page :
              history.push('/');
            • f- Catching the returned and printing it using the default method using custom message :
              console.log(`Error: $(err.message ) ` )

        • (c) Modify {Delete cetain item } third CRUD operation => {handleDelete() } function responsible for deleting /removing a certain item from the assigned database in the [posts] , by using {axios} method by following the next steps :
          1. 1- before the try {} :

            Adding the {async} to the main fucniion of the deletion {handleDelete()}

            Adding the [id] paramater , to be obtained from the user component -> to be used to detect the curernt item to be deleted


          2. 2- inside the try {} :
            • (a) Direct Awaiting the delete crud operation from [api] - from the full path of baseURL + id of hte current element post - [no need ot define a varialbe ] , by using the next crud code :
              await api.delete(`/posts/${id}`);
            • (b) Define a new list of posts (after excepting the deleted item ) that including all items except the current filtered deleted post - according to the current item by id , by using the (filter) method :
              const postList = posts.filter(post => post.id !== id) ;
            • (c) Setting the new post filtered list inside the post state [which will be displayed in the main home page] :
              setPosts(postList);
            • (d) use the defined useHistory hook variable to route into a certain route [home page] :
              history.push('/');
          3. 3- inside the catch(err) {}

            catching the returned error by using the custom error message :
            console.log(err.message);


        • (d) Modify {Modify/Edit} fourth CRUD operation => {handleEdit() } :
          1. 1- Before try {} :
            • a- Adding the (async) to the main function of handleEdit()
            • b- Adding (id) parmater to be used for checking of the current item to be modified and sent to the assigned api by using {.put() } , and as one of the modifed item object properties
            • c- Define a variable of date -current date- => [as a one of the modified item object's property : dateTime] usijng the defined and imported { format } from 'date-fns' , with custom format pattern
            • d- Define a variable of [full updated/modified item post object] after being updated/editing [with assigning both of [title] & [body] with defined status ] , as next code :
              const updatedPost = { id, title: editTitle, dateTime, body: editBody };

              // the modifed object properpties as following -inorder- :
              [id] : represent this item(post) id , will be assigned by incomming paremeter of the main function
              [title] : represent this item(post) title, will be assigned by the value of the {editTitle} prop
              [dateTime] : represent this item(post) current date [date of creation] , will be assigned by the value of the upper defined vcarilge {dateTime}
              [body] : represent this item(post) body , will be assigned by the value of the {editBody} prop

          2. 2- inside the try{} :
            1. a- Define a 'response' awaited variable of getting the updated full object curd operartion , to send the edited/modified values into the assigned api path by using the {.put} method , as following :
              const response = await api.put(`/posts/${id}`, updatedPost);

              // we here used the {.put} method due to we will send a full edited/modified item object into the assigned api path wihth sub dynamic path of this item id , but incase of editing single property of the item -> we must use the {patch} , and used the next parmater ;
              [api dynamic path ] => '/posts/${id}'
              [the full object of modified item ] => {updatedPost}

            2. b- Assign/replace the updated item object value into api datasource [posts] , by using mapping - because we will not add the old data of the posts - :
              setPosts(posts.map(post =>
              post.id === id ? { ...response.data } : post
              ));

              // we here set a modifed post/item inside the datasource database after condional confirmation of the current item id is equal identically with the id pamreter
              // the assinging after getting confirmed is by assign the seperated [response data] object of the new modified variable , or assign the current object post item as it is

            3. c- Setting empty values of both [post body] & [post title] after being set wihtin the defiend full object [updatedPost] :
              setEditBody('');
              setEditTitle('');

            4. d- Re-directing to the home page after accomplished modifying/editing crud orepration ,using the imported history('/') to the main home page route :
              history.push('/');
