Main Tools & Libraries requried for building the [Login Form] :

This [Login Form] is about a Login Form with using the folowing tools :

  1. [Axios] library for asynchoronous responding process [post]
  2. {Context} : for define the [Authentication Provdier function] which will include the main Authentication coinfguration settings to store the main inserted data of the [Login Form]
  3. [node.js server] + [Mogo DB] connection -> [available in the Node course ] to provide the main endpoints of server's [response data] & [response accessToken] & [response roles]



B] main steps syntax of { Login Form + Submit } app :


the syntax , as following :

  • 1] Required Importing :
    1. a- Basic react state importings => { React, useState, useRef, useEffect, Fragment }
    2. b- [axios] library => for dynamic respoding to the assinged [api] and posting process

  • 2] Required Defintions and variables [ BEFORE the main functional component {Register()} ] :
    1. a- Validation variables definition :
      1. Regular expression variables reqrured for validation of both of [username input] & [ password input] (the matched input will be validated by custom function )

    2. b- Required api sub path {LOGIN_URL} variable :

      The sub path value variable of register => be used in posting procees => [as extension with the after main {baseURL} of defined api ]

  • C] Required Defintions and variables [ AFTER the main functional component {Login()} ] :
    1. {userRef} => Define a [useRef] of username input ref [to be used in set focus on the uesername input field when page is being load] => [ wintint the {ref} attriutes ] ]
    2. {errRef } => Define a [useRef] of error meassage as a 'uerRef' [ to be used in focus on the error paragraph of when it is occured ]
    3. Define group of variables [states] for controling [username] input field funtcions :
      • [user, setUser] empty string value -> state to put the inserted username input field value
      • [validName, setValidName] false boolean value => state to set validated value of the username input field value [within the valdatiing useEffect method ]
      • [userFocus, setUserFocus] => false boolean value => state to set validated focus state value of the username input field value [wintin the both of [onFocus] & [onBlur] ]

    4. Define group of variables [states] for controling [ password ] input field funtcions :
      • [pwd, setPwd] empty string value -> state to put the inserted password input field value
      • [validPwd, setValidPwd] false boolean value => state to set validated value of the password input field value [within the valdatiing useEffect method ]
      • [pwdFocus, setPwdFocus] => false boolean value => state to set validated focus state value of the pasowerd input field value [wintin the both of [onFocus] & [onBlur] ]

    5. Define group of variables [states] for controling [ matchedpassword ] input field funtcions :
      • [matchPwd, setMatchPwd] empty string value -> state to put the inserted confirmation pasword input field value
      • [validMatch, setValidMatch] false boolean value => state to set validated value of the confirmation password input field value [within the valdatiing useEffect method ]
      • [matchFocus, setMatchFocus] => false boolean value => state to set validated focus state value of the confirmation pasowerd input field value [wintin the both of [onFocus] & [onBlur] ]
    6. Registration Error handling state variable [incase of the Submit error is existed ] :
      [errMsg, setErrMsg] : to handle the responded [error] message - > within serveral useEffect methods , conditional rednerings
    7. Define submiting [state] successful registration handling [incase of the submit secceded ] :
      [success, setSuccess] => to be set by inital value of [false]
    8. Define the focusing method [useEffect] on the username input by using useEffect [ when the component is page is loaded for the first rendering]
    9. Define the [username input field Validating useEffect method ] + setting the [user state] as the dependecy of useEffect [to respond each change]
    10. Define the [[password] & [Checked password] input fields Validating method] + setting the [user] & [matchPwd] states as the dependecies of useEffect [to respond each change]
    11. Define a method of setting an empty value for error message while of changing in any of states of the Form inputs [using the {setErrMsg} state setter function wiht using all of the states representing the inupt fileds [user, pwd, matchPwd] ]
    12. Define the {handleSubmit} method with adding anti-hacking checking [incase the user pased the validation processes of one of input(s) fields] , including to the next steps :
      • a- preventing default method :
      • b- Adding anithacking checking procedures [to prevent enabling submit button before validation ] , including :
        1. 1- Redefining the both of [regex validation] variables [user , password]
        2. 2- Checking for above defined [Regex validation] variables is not existed [means the validation has been passed ]
      • c- [main Async - await respondging to the assigned api ] using the {try-catch} block , contianing the next steps :
        1. {try } : Define the responding variable of [post] process with api using the api link of [REGISTER_URL] , with assign of all input fields states values [input states] inside the (stringfy) mehthod paremeters , as following :

          1- reponse variable :

          • [user] parameter => the username input state
          • [pwd] parameter => the password input state
          • {} method => custom object of the post => include [header] property & [ withCredentials] property


          Priniting the value of returned form response [reponse.data] & [response.acessToken]

          Setting the state of [success] state by true - incase of sucessful registartion submting - , to be used for rendering display

        2. {catch} : including (2) main stepes :
          • 1- displaying differnet custo error message accorgding to the responded types of error types , as folowing :
            1. NO catched Error => [the {err} catch paremeter retuned null ]
            2. {409} error type => the username has been taken by anther user
            3. Any other type of public error =>
          • 2- Focusing on the generated error of [error paragraph] with in the script , using {errRef} that asigned inside this paragraph
  • D] main return of the functional component :

    using conditional rendering [according to {access} defined state ] to display one elemtn of the following :

    1. First rendering [log-in link ] to log-in page => dispalyed after the succesful registarion process -> to prvodie the login for the user
    2. Second Refndering [Registration Form ] => the first and default element to be displayed to the user to provdie the registration first and [sign-in] link , and including :
      1. 1- [error paragraph] => Displaying the error message if it is existed (before the main form ) , by usign conditional classname values [hide ,display ]
      2. 2- [Registration Form] => containing next several sections :
        • a- [Username input field] section =>
