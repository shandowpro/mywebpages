29 React Router v6 in 20 Minutes RRv6 Upgrade Refactor Tutorial concept :

A] main concept and some about the npm packages versions properties :


  1. Main concept about npm packages versions :
    • As a public syntax rule in the library wihtin in npm , as following :
      v : major version . minor version . package number
    • to update the next major version => we need to reinstall the whole library again wiht the new verion via using the npm , as next code :
      { npm i react-router-dom@v6.0.2 }
    • to update the next manior version or next packate of the cuyrenby major version => we need to just to update the current verinosn of library , as next code :
      { npm update react-router-dom }



B] main syntax steps to converting the {react-router-dom} from (v5) - (v6.0.2) with in refactoring the [blog react] app :


Due to we will now update the major version of the [react router dom] library , we will need to refator all the app , with the following step :

  • a] Inside the {index.js} syntax modifications needed the as following procedures :
    1. 1] Modifications in imoprtings , as the next code :
      // importing the [routes] instead of [switch in the App.js]

      import { BrowserRouter as Router, Routes , Route } from "react-router-dom";



    2. 2] Modificaions on the main render fuction :
      ReactDOM.render(
      < React.StrictMode >
      < Router >
      < Routes >
      // this is the public route of the App [ we must add * after the slash + replcace the 'component' by 'element' ]
      < Route path="/*" element={ < App / > } >
      < /Routes >
      < /Router >
      < /React.StrictMode > , document.getElementById('root')
      );


  • b] Inside the {App.js} syntax modifications needed as the following procedures :
    1. 1] Modifications in imoprtings , as the next code :
      // a- importing a new main parent wrapper component from {Layout.js} file to e used here :
      import Layout from "./Layout";

      // b- Replacing importing {useNavigate} instead of {useHistory} :
      import { Route, Routes, useNavigate } from "react-router-dom";



    2. 2] Modificaions and definitions inside the component and before the return :
      // a- Define a state of [{navigate} extracted from {useNavigate}] instead of {history} exteacted from {useHistory } :
      const navigate = useNavigate();

      // b- Replace the order of path navigation of {history.push('')} by { navigate('') } [in all project files] :
      const navigate = useNavigate();

      // c- Modifications inside the return :
      // using the [Routes] as the overall parent element of all other routes elements [niot accepting any upper codes even comments ] :
      < Routes >
      // Define the main parent Route element of the imported {Layout} component with all it's props :
      < Route path="/" element={ < Layout search={search} setSearch={setSearch} /> } >
      // This is the [First main route] of the all application [which has 'index' property] and represent the home page
      < Route index
      < element={ < Home posts={searchResults} fetchError={fetchError} isLoading={isLoading} / > }
      />


      // This is the [Second main route] of the all application [which has only 'path' property] and include several inner pathes :
      < Route path="post" >
      // 1- The inner index route of the second route 'post' :
      < Route index
      element={ < NewPost
      handleSubmit={handleSubmit}
      postTitle={postTitle}
      setPostTitle={setPostTitle}
      postBody={postBody}
      setPostBody={setPostBody}
      /< }

C] VIP Notes about using [ v: 6.0.2 ] hook :

  • All defiend routes must be wrapped by major route of Layout element
  • First route to be defined must be the home ruote that has the 'index' prop , and represent the path of '/'
  • We can define a parent route that includes another several routes
  • The sub-router may include and inner route that has it's 'index' , which represent this sub-router home page
