15 React Beginners Challenge Fetch API Data React JS Beginners Project concept :

A] Main project concept , and features :


Fetching a real data from the given api [ jsonplaceholder.typicode.com] and displayiong the obtained data in a UI table applciation , by using following tools :

  • a- The main [App()] will include (2) main [lines] of components that represented s the following :
    1. {Form} -> include a main {form} elememnt that contain (3) imported reusable components [Button] elements -> which inclde the main tabs buttons with logic to shifting data display
    2. {Table} -> include the part of the [in-depth data formated displayer] elements and it's components

  • b- The main [api url] wil be defiend as string value inside a variable : 'https://jsonplacaeholder.typicode.com'

  • c- Main [api main cruds operations execute] logic procedures :
    1. 1- main logic functions and steps for each component =>
      • 1.1 : {App.js} =>
        1. [components] : Include the main parents components {Form} & {Table}
        2. [logic] : Include main defintions of each following :
          • Define a variable of [api url] to be used to call the desired api url provider
          • Define a state to set the request type value as sub api url [users , comments , posts ]
          • Define a [data container] state as array of items to be used as container of obtained data from the api url
          • Define the {useEffect} hook to deal with data from the given api url + and assign the [reqType] state as a useEffect hook main dependency , and will include the following logic :
            1. {fetchItems} => Define the function structure responsible for :
              • 1- Connect with defined Api url using fetch() method with given api [main fixed url + sub dynamic ]
              • 2- Calling data from the given connected Api url
              • 3- Assign the obtained data inside the defiend [data container] state
              • 4- Catching Error incase of not responding conoection and display the error text

            2. Executing the defined function above {fetchItems()} to be able to work inside the useEffect hook
        3. Returned :
          1. First component [Form] => will assign the props values of both :
            • {reqType} => will only assign it's definition sturcture as a state varialbe , but it's value will be assigned inside the child componenet {Button : inside the button element : [onClick()] event with the value of another prop {buttonText} }
            • {setReqType} => will only assign it's definition sturcture as a state setter function , but it's value will be assigned inside the child component Button : inside the button element : [onClick()] event with the value of another prop {buttonText}
          2. Second component [ Table ] => will assgin the value of [assign here ] :
            1. {items} -> the [data container] state array => will be gain it's value from the deinfed {fetchItems} while cionnecting with given {apiurl} wihtin the useEffect

      • 1.2 : {Form.js} =>

        This is the first main parent component [App] of the main first line [{App.js} => {Form.js} => {Button.js}] , with following properties :

        1. 1- This component is about a form element , with deactivate the default submiting feature => by using e.target.preventDefault()]
        2. 2- [props] : This conmpnent will recieve props (by calling it inside it's parent component) of both [reqType, setReqType] will be used inside each called [Button] component , to be assigned when being called inside the main {App} component
        3. 3- [Returned] : {Button} a reusable component will be used as serveral called elements with props of { buttonText : to be assinged here & [reqType, setReqType] : to be assigned inside the main App component }
        4. 4- [logic ] : no logic needed here

      • 1.3 : {Button.js} =>

        This is the second component of the main first line [ {Form.js} => {Button.js}] , with following properties :

        1. 1- This component is about a unified [button] element
        2. 2- [props] : This component will recieve props (by calling it inside it's parent component) of both [buttonText , reqType, setReqType] will be used inside each called [Button] component , to be assigned when being called inside the main {Form} component
        3. 3- [Returned] : unified structure of the jsx [button] with logic functions and props to be when calling this element inside the it's parent Form
        4. 4- [logic ] : no logic needed here


      • 2.1 {Table.js} =>

        This is the First main parent component [App] of the second line [ {Table.js } => {Row.js} => {Cell.js} ] , with following properties :

        1. This main component represent the second section of this app , which is in-Depth data displayer as a table form
        2. The direct parent component for this component is -> [App.js]
        3. This component will recie the main data conetainer {itmes} from it's parent [App.js]
        4. [props] -> {items} is the main data container to be assigned inside the main app by useEffect hook react by using fetch()
        5. [return] -> a {table} JSX element -> {tbody} JSX element - > include a mapping of {items} prop of the child imported component {Row} reusable component imported

      • 2.2 {Row.js } =>

        This is the second component of the second line [ {Table.js } => {Row.js} => {Cell.js} ] , with following properties :

        1. The direct parent component for this component is -> [Table.js]
        2. This component will return the recieved single data entry {item} and stringify it into key and value for each single data item as a imported component [ Cell ]
        3. {item} prop : will be used inside the parent component [Table]
        4. {Cell} Imported child element : will be used inside here
        5. {return} => include a html [tr] element that contains Mapping through the imported prop [item] => as an object , and destructuring it as a key and value [key, value] as props of the imported {Cell} as following :
          1. key : the mapping key value
          2. cellData : the stringified values of returned [value] of object entries of (item) prop to be assineed inside the direct parent component [Row]

      • 3.2 {Cell.js } =>

        This is the third component of the second line [ {Table.js } => {Row.js} => {Cell.js} ] , with following properties :

        1. Single piece of item's row as a cell of data , include => [cellData] prop : will be assinged directly inside the it's parent component [Row] -> as the stringify value of the object parmater [value]

C] VIP Notes :

  1. To destruct data comming from a the fetched given [api url] we need to stringify the json to be able to read as an object , and there (2) types of method to deal with the json data format :
    1. {responseVar.json()} => to convert the comming data from the database of the given api -> to be able for reading inside a variable
    2. {Object.entries(item).map( ([key, value]) )} => to deal with single incomming data entry item as an object and destruct it into [key] and [value] as the object proeprties
    3. {JSON.stringify(value)} => used to convert the value property of incomming json single entry data format into string format to be able for display
