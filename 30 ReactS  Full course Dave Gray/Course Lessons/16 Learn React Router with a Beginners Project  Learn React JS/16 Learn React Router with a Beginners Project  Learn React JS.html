16 Learn React Router with a Beginners Project Learn React JS concept :

A] Main project concept and features and properties of this application :


This application is like a bloging web blog application , to add a new pblog and all other blogs

This application wil use teh next libraries :

  1. [react-router-dom] (v:5.2.0)

This application will have the next features :

  1. Links buttons in the header to access into the web application inner pages routes
  2. Home page to display all blog wiht public details
  3. Inner page to display each blog in-details by uing it's id
  4. Ability to delete any of blog within each in details inner page
  5. Inner page provides the Ability toadd new blog


This application will include (2) types components , as following :

  • a- Fixed components {3} [main sections in the main home page ] , following :
    1. {Header} -> The header sectjon of the every page
    2. {Nav} -> The navbar the including iner pages buttons
    3. {Footer} -> The footer section of every page

  • b- Routers inner pages components {4} [Routes of inner pages of the app ] , following :
    1. {Home} -> The main landing page
    2. {About} -> The about page
    3. {NewPost} -> The page including a form of adding new post
    4. {PostPage} -> the page of dispalying all post details

B] Practical steps and syntax and method of creating the project with [react routering dom] library :


After project installation using npx , we have to execute the following steps :

  • a- Installing the react router dom library using the next code :
    npm i react-router-dom -S => this will install main s library version , but to install the [v: 5.2.0] we can edirt manulaly and re-install the npm
  • b- Modifying {index.js} file , as following :

    1- Importings :

    // importing the required classes and hooks for the react routering matched with [v:5.2.0] : import { BrowserRouter as Router, Route } from "react-router-dom";

    2- Modifying the syntax as next code :

    ReactDOM.render(
    < React.StrictMode >
    // Wrapping the App main component wihtin the [Router] react router dom library hook :
    < Router >
    // Replace the App calling with Route and adding the App inside attributer of component :
    < Route path="/" component={App} / >
    </ Router >
    </ React.StrictMode > , document.getElementById('root')
    );



  • c- Modifying component [App.js] :
    1- Importing fixed components [sectinos components] :
    import Header from './Header';
    import Nav from './Nav';
    import Footer from './Footer';

    2- Importing inner page components [ Routes components] :
    import Home from './Home'; // the main Router page
    import About from './About'; // the About Router page
    import NewPost from './NewPost'; // the new post adding Router page
    import PostPage from './PostPage'; // the detailed post Router page
    import Missing from './Missing'; // the 404 Error Router page

    3- importing the required classses and hooks for the routering => [according to v:5.2.0 ] :
    [Route, Switch] -> will be used in this lesson , but [useHistory] -> will be used in next lesson
    import { Route, Switch, useHistory } from 'react-router-dom';

    4- Define the main App using the imported components wihtin the react routering library :
    function App () {
    return (
    < div className="App" >
    < Header />
    < Nav />
    all Routes should be inside this class to activated as routes :
    < Switch >
    // Define a router for inner page Route of main [home page] route -> [must include exact attriubte in this version ] :
    < Route exact path='/' >
    </ Home >
    </ Route >
    // Define a router for inner page Route of child [New Post] page route -> [must include 'exact' attriubte in this version , and the path must be all in small letter ] :
    < Route exact path='/post' >
    </ NewPost >
    </ Route >
    // Define a router for dynamic inner page Route of child [New Post] page route -> [must NOT include 'exact' attriubte - due to it is a sub router of the NewPost: post , ansd usign id for dynamic routering - , and the path must be all in small letter ] :
    < Route exact path='/post/:id' >
    </ PostPage >
    </ Route >
    // Define a router for child component [About] route -> [must defiened as inline code of route , due it is a route for a component not for inner page ] :
    < Route path='/about' component ={About} />
    // Define a router for child component [Missing] route for 404 error page -> [must defiened as inline code of route , due it is a route for a component not for inner page + with path value of '*' to be redirected to this page incase of non defiend path ] :
    < Route path='*' component ={Missing} />
    < Switch />

    < Footer />
    </ div >
    )
    }






C] VIP Notes :

  1. To destruct data comming from a the fetched given [api url] we need to stringify the json to be able to read as an object , and there (2) types of method to deal with the json data format :
    1. {responseVar.json()} => to convert the comming data from the database of the given api -> to be able for reading inside a variable
    2. {Object.entries(item).map( ([key, value]) )} => to deal with single incomming data entry item as an object and destruct it into [key] and [value] as the object proeprties
    3. {JSON.stringify(value)} => used to convert the value property of incomming json single entry data format into string format to be able for display
    4. main path optinos can be uesd as value of [path] attribute of the [Route] , as following :
      1. '/' -> represent the [Home page] => will rourt to the main page
      2. '*' -> represent the [Error page] => will rout to error page of [404] when user using undefined route value
      3. '/routeName' -> represent the specific route name of a certain [page] => will rout to the assinged route comopkjet
