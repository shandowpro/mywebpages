2 Setup and Installation concept :

{Node.js} Runtime Environment =>

  • It is the run time environment , which can run JS ouside the web browser
  • The {ReactJS} is using Spciasl lnagusage {JSX} & {TSX} as merged languages , with {.jsx} & {.tsx} extenison file to write react code ,which will converted to the JS , after transpilaition
  • {node.js} official source to download webiste => [nodejs.org/en ] -> for {windows X64} -> version {18.16.1 LTS} to be compatable with react node and other runtimne and environemnt , and fully tested

{NPM} Node Package Manager =>

  • This package is auto installed with node.js
  • This is the offical manager to download react.js libraries and deal with any package run with {node.js} , such as [install any pack from {react.js} whether if this pack from offical site or contribution from any developer] , such as
    { npm -install packName }

{NPX} Node Package Executer =>

  • This package is auto installed with node.js
  • This package is responsible for executing and running the reactJS packages within node.js environemnt , such as
    { npx create-next-app }

{Boiler Plate} Framework Documentation =>

  • This is the iofical documentaion of the reactJs framework by using {boiler plate}
  • From the offical web site => [react.dev] => will provide critcal information about using the {boiler plate}
  • [Boiler plate] -> will make the react app configuration much more auto and dynmaic , and {reactJs} instalation
  • [Boiler plate] -> will recommend to use one of the folllowing frameworks :
    1. {next.js} framework , which is the most recommended framework to use a react.js fullstack for web applications and web sites
    2. {remix.js} framework , which is a react.js fullstack for web applications
    3. {Gatsby.s} framework , which is a react.js fullstack for web applications
    4. {expo.js} framework , which is a react Native fullstack for Mobile applications

{Next.js} Framework =>

  • The most recent and famous featuristic fullstack framework to be used and recmmended to used with node.js runtime

Setup configuration {ReactJS App} using {Boiler Plate} , according the following procedures =>

  • Check for each {node} version by using following command
    { node -v } => [for checking for node runtime ] { npm -v } => [for checking for the node package manager ] { npx -v } => [for checking for the node package executer ]

  • Open the project path with any command line Terminal , and using the folllowing command line:
    { npx create-next-app } =>
    {if this is your first time to establish react app => will start downloading the [next.js] in your pc }
    {other wise => will start establishment application procedures }

  • Starting inserting the react app configuration , as following steps :
    • insert react application name , according the following rules:

      not including any spaces

      can including {_} || {-}

      can be just as default name [my-app]


    • choose to use {typescript [TS]} with in the react application :

      [yes] => it is recommended to use [TS] (which is a super set of the js) , due to solve the weak variables types of the JavaScript , by providijng strong stongly typed for each variables and function results

      [no] => not recommended


    • choose to use {ES lint} with {Typescript TS} in the react application :

      [yes] => it is recommended , to provide auto checking for application while writing the codes and before compilation

      [no] => not recommended


    • choose to use {Tailwind CSS} {NO} =>

      will not be used in this course

      {Tailwind CSS} => It is a strong and much easier CSS framwork more than bootstrap , with the same concepts of the bootstrap

      {Tailwind CSS} => it is with the same concepts of the bootstrap , but with simplifying these conepts to {CSS utilities} with more logic and easier more than bootstrap Framework


    • choose to use {'src'} directory {NO} => due to our project in not big

    • choose to use {App Router} {Yes} =>

      {App Router} => This concept is existed in any ReactJs App [website || webapp] , and basesd in {Virtual Dom}

      {App Router} has (2) types =>

      • {App Router} => which will be used in this application
      • {Page Router} => which will be studied later

      {Virtual Dom} => It is about {one HTML Element} method

      {one Html Element } => It is about starting the react app with one main element , which will rendering all other elements within website {inner pages}

      Router app is running by hte node.js witin following :

      • Get component
      • Execute rendering
      • put the elements in the right place

      The router app mechanism :
      when the page is getting navigated to another iner page , the root {main HtTML element} component is calling the other page single conponents , with in the same page


    • Customize the {default import aliase} =>

      {Yes} => will use (@) before hte imported component

      {No} => will not use this option


    • Starting initailizing the app =>

      Which is the feature of using the {boiler plate} , which is instralling hte configuration of hte react app automatically , with out need to the mannauly configurations
