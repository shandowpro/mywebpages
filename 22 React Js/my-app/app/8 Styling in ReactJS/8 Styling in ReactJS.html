8 Styling in ReactJS concept :

{ Styling in ReactJS } concept with including conponents :

  1. To use the {CSS codes} to format the elements and component code within the {ReactJs}


{ Styling in ReactJS } importance and usages with including conponents :

  • It is essntial the elements and component within ReactJS
  • There are several methods to use Css Format codes [simmiler methods to HTML ] within ReactJS , but by diffrent syntax


{ Styling in ReactJS } methods :

  • [Importing CSS Files & ClassName] method :

    It is about importing the css files Modules from the ['app'] directory , and using one of the class names existed imported file with in the current rendering page script

    In the ReactJS there is a specailized CSS Module file for each Rendering page , such as [for {page.tsx} => there is {page.module.css} within the page directory 'app' ]

    In the ReactJS it is requiring to create seperated file Module for each new page

    In the ReactJS there is a Global CSS file , for shared CSS codes between all project pages files

    All imported CSS files modules will be called by using unified keyword 'style'

    The {Class Name} used to format the element/component, must be used in the imported file

    The syntax to import the CSS file as following :
    // Import main Rendering page css file [page.module.css] :
    import styles from './page.module.css' ;

    // Import main Rendering general css file [globals.css] :
    import styles from './globals.css' ;

    import styles from 'page.module.css' ;
    // Calling One of CSS Classes within the included element :
    function Home() {
    return (
    < main >
    < div className = {style.className} >
    < /div >
    < /main >
    ) ;
    }


  • [Inline CSS Codes] method :

    It is about to write the CSS codes directly within the included element, just like in the HTML , but with different synatx

    The Syntax of using {Inline CSS} Method as following :
    // Writing Css codes directly within the included element :
    import styles from 'page.module.css' ;
    function Home() {
    return (
    < main >
    < div { { style = "color:red , backgroundColor:white" } } >
    < /div >
    < /main >
    ) ;
    }





Vip notes about using CSS codes within the ReactJs elements :

  • In the {ClassName} method => the class name must be existed in the imnported CSS file
  • In the {Inline} method => the style attribnute must be with value with curly Brackets , and being deal as an Object , that mean to follow the next rules :
    1. Using double Curly Brackets to assing the value of style Attribute
    2. write the css property's Name by [camel case] incase of multiple word CSS name property
    3. write the css property's Value by within single or double Quotes
    4. Using Comma ',' instead of using ';' => to seperate between each property and another inside the assigned CSS properties
  • In the {Importing CSS files with using ClassName} method => :
  • To import any Css file => must follow the next rules :
    1. Use the {styles} keyword, within the import code
    2. Use the {./} prefix before the path of imported CSS file

  • There are (2) kinds of imporoted files :
    1. {Certain specific file for the current page} :

      Which is determined by the current page file, such as: [page.tsx] => {page.module.css}


    2. { Public file shared for the all project pages files} :

      Which is determined by the current page file => {globals.css}


  • To use the [ClassName] within the element => must follow the next rules :
    1. Must write by using camelCase [className] instead of [class]
    2. The [className] value must be inside a single curly brackets { } and using 'styles' keyword with extension of the calss name existed in the improted file
