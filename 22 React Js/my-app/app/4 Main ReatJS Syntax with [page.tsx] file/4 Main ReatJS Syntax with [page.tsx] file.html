4 Main ReatJS Syntax with [page.tsx] file concept :

{page.tsx} main Default function of main component [with using next.j framework] =>

  • Using reactJs with [Next.js] is requiring {Rout} dierctory that holding the project files for each page => which is by default with main page is folder {' app => page.jsx '}
  • For any {inner page} in reactJs with [Next.js] is requiring to create new {Rout} dierctory that holding the poject files for each inner page , with folwoiong properties
    =>
    1. This directory will be by name of this innerpage
    2. This directory will include whole files same like in the main [app] directory of main component


{page.tsx} main sections =>

  • {page.tsx} -> [Importing] section :
    1. This section where the current project will import the requierd [Modlues] and [packages] from other inner/exteranl folders to be used in this project

  • { Home() } main Default function of main component :
    • This is the main function to render the whole components the of main project
    • This component function should be with name of the page => [which is Home() in the main Home page ]
    • {return()} => has the following properties :
      1. This is the return of the main component default function
      2. This will render all components of the page/project as children components
      3. Inside this return we will write [reactJS] codes using the {JSX} || {TSX} language
      4. This {return()} will include the following components :
        • {< main > } => which is only for the main component, and will contains all elements written by HTML and JS/TS {JSX}|| {TSX} , with following properties :
          1. apperantly it is similer to the HTML Elements syntax , but with the unique code of the [JSX] || [TSX] which will render the [JSX/TSX] components from the {reactJS} packages and Modules
          2. [JSX/TSX] is different from the original HTML and XML and JS , such as an Example :

            [JSX/TSX] is using 'className' with in the element attribute instead of using 'class'

            [JSX/TSX] is using ' { ModuleName.className } ' with in the element attribute's value instead of using 'direct classname'

{Child function component} :

  • It is a child non main component , that will be included within the main function , and has the following properties :
    1. Must has { return() }
    2. Not has { < main > } inside
    3. can be inlcuded inside each other and inside the main function

Addition features with in {ReactJS} with in live preview project :

  • The {ReactJS} with using {Next.js} is providing {Live preveiwing} of the current project (to preview the codes effects with out nedd to reload ) by providing {local host sever} , by using terminal command :
    { npm run dev } => generate a {local host} of the current project for live previewing your project]
  • by using "Live Preview" => any changes within your project codes will be directly seen in the oppened project page by generated {local host} once the codes is saved in the script
  • Also there are farther additinoal feature of {auto save} can be used by your code editor => [which will auto save your codes wihtout need to save]

Practical script of [main component function] & [children functions] :

  1. 1] Create the main component function :
    // Define the main function of main component {Home() page } [with using best practise of using default at last of the script] :
    function Home() {
    // using return to include all page elements :
    return (
    < main className = {style.main} >
    < p > Hello world < /p >
    < /main >
    ) ;
    }

    // Exporting the main compoonent function at the last of script [best practise] :
    export default Home ;



  2. 2] Create Multiple children components functions & Include them in the main component function:
    // A] Define the first child function component {Header() component function } :
    function Header() {
    // using return to {include just only one page element} [with out using main element ] :
    return (
    < div >
    Helloo from the header of the page
    < /div >
    ) ;
    }


    // B] Define the second child function component {Description() component function} :
    function Description() {
    // using return to {include just one page element } :
    return (
    < div >
    Hello from the header of the page
    < /div >

    < div >
    Hello from the header of the page
    < /div >
    ) ;
    }


    // C] Define the main child function component {Home() component function} with adding defined children components functions :
    function Home() {
    // using return to include all page elements & defined children components functions :
    return (
    < main className = {style.main} >
    < p > Hello world < /p >
    < Header/ >
    < Description / >
    < /main >
    ) ;
    }

    // Exporting the main compoonent function at the last of script [best practise] :
    export default Home ;



  3. 3] Create Multiple children components functions & Include them inside each other [nesting] and then inside the main component function:
    // A] Define the first child function component {Header() component function } :
    function Header() {
    // using return to {include just only one page element} [with out using main element ] :
    return (
    < div >
    Helloo from the header of the page
    < /div >
    ) ;
    }


    // B] Define the second child function component {Description() component function} [Including the Child other first defind function component (Nested) method ] :
    function Description() {
    // using return to {include Multiple page elements and other child function } :
    return (
    // using {ReactJs Fragment} method to [include more than one element || include other child component function] :
    <>
    < div >
    Hello from the header of the page
    < /div >

    // Including a defined child component function [Header()]
    < Header/ >
    </>
    ) ;
    }



    // C] Define the main child function component {Home() component function} with adding defined children components functions :
    function Home() {
    // using return to include all page elements & defined children components functions :
    return (
    < main className = {style.main} >
    // Including page Element :
    < p > Hello world < /p >

    // including {Header()} defined conponent function
    < Header/ >

    // including {Description()} defined nested Component function
    < Description / >

    < /main >
    ) ;
    }

    // Exporting the main component function at the last of script [best practise] :
    export default Home ;
