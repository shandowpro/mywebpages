9 Routers and Routing in ReactJS concept :

{ Routers/ Routing and Link } concept in ReactJS :

  1. To navigate throught the project's pages without need to reload the current page , by using component conept of the reactJs
  2. Router in ReactJs is attained by using the only home() component function in [pate.tsx] , by replacing the other's inner page comnponet function within the home page conponent function
  3. Router in ReactJs can be used with the [link] keyword as hyperlink element, instead of using <a> with using ['href'] attribute


{ Routers/ Routing } Requirments and Procedures in ReactJS

  • This could be using by following procedures and Rules :
    • Each Inner page files must be in sepertated directory/folder => inside the 'app' folder of the project
    • This folder will include all inner page component files
    • This folder will be used to replace the inner page inside of the home page
    • Each Inner child page's directory must has the following files :
      1. {page.tsx} => will include the following :
        1. [Link] Importing => [incase of sing the links inside this inner page]
        2. [child component function ] => [which will include the links in upon desired]
        3. [Default Exporting of the child component fuction ]
      2. {page.module.css} => which will be used to set any CSS properties specified for this inner page

    • { Routers/ Routing } conceptual Mechanisms and comparison between [old-reactJS and new-nextJs-] in ReactJS :

    1. {old-reactJS} origin reactJs only =>
      • This method is requring to install special libraries and setting certain configuation
      • This method is NOT recommehnded in recent reactJS projects , due to probable confliction will be occur
    2. {new-ReactJS with NextJS} =>
      • this method is NOT requring to install special libraries or even certain configuration
      • This method is strongly recommehnded in recent reactJS projects , due to simplicity to use , by using only bliult-in NextJS fuction

    { 'a' vs 'Link' } Mechanisms and comparison between [old-reactJS- and new-nextJs-] in ReactJS :

    1. {Link in the new nextJS } method => To use the [Link] element inside any conponent page , must follow the cetatin procedures and Rules :
      • [import] in each Inner page component function is a must , with unfied keyword and path :
        1. unified {keyword} => Link
        2. unified {path} => 'next/link'

      • [Link] element syntax [the new recommended method] :
        // 1] Creating a link in the {Home page} , TO navigate to the {about page} using new [ Link ] method
        // General importing code for any link in the next directory
        import Link from 'next/link' ;
        function Home() {
        return (
        < main >
        // The navigate link from {home page} -> {about page} :
        <Link href = "/about" >
        Go to about page [ using Link ] method
        < /Link >
        < /main >
        ) ; }

        // General exporting code
        export default Home ;

        // 2] Creating a link in the {about page} , TO navigate to the {Home page} using [Link] method
        // General importing code for any link in the next directory
        import Link from 'next/link' ;
        function About() {
        return (
        // The new method code of navigate link from {about page} -> {home page} :
        <Link href = "/" >
        Go to about page [ using Link ] method
        < /Link >
        ) ; }

        // General exporting code :
        export default Home ;
    2. To use the [a] element inside any conponent page , must follow the cetatin procedures and Rules :
      • There is no to import any files from the Next directory , due the browser will use the traditional method to Re Rendeder/Reload the page
      • This is not the ideal method to navigate thorugh the project's pages , due it will return the server and Re-load the whole page , which is the best proctical method and take much longer period
      • [a] element syntax [the old NOT recommeded method ] :
        function Home() {
        return (
        < main >
        // The new method code of navigate link from {about page} -> {home page} :
        <a href = "/about" >
        Go to about page [using old method]
        < /a >
        < /main >
        ) ; }

        // General exporting code
        export default Home ;
