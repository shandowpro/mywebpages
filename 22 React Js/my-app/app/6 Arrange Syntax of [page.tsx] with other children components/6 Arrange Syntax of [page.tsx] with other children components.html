6 Arrange Syntax of [page.tsx] with other children components concept :

{ Arranging reactJs project } concept of handling tree folders files [with using next.j framework] :

  1. To create specalized file for each component function , and then export each one to it's parent , for more arranged and specialized files tree

{ Arranging reactJs project } steps :

  • Create container parent new folder to hold all children files inside , by recommended name of ['components']
  • Create new folder for each child component function by name of the defined child component , which will contain the new created file
  • Create new file for each child component function by name of the defined child component
  • Arrange each new created file as following orders :
    1. A] {Main file || [app/page.tsx]} => should contain following contents by order :
      1. Importing each child component file by it's path in the project files tree
      2. Define the [Type object] that used to defein the main function [Home()]'s props datatypes -> [ incase of defining props in the main function ]
      3. Define the [main function]
      4. Assign the value of the passed props from the imported files
      5. Exporting the defined component function as a default function => [to able to read from the imported file ]


    2. {sub main child file || [subMainPage/subMainPage.tsx] } => should contain following contents by order :
      1. Importing each child component file [that been used in the current sub main file] by it's path in the project files tree
      2. Define the [Type object] that used to defein the main function [Home()]'s props datatypes -> [ incase of defining props in the main function ]
      3. Define the [Sub main function]
      4. Assign the value of the passed props from the imported files
      5. Exporting the defined component function as a default function => [to able to read from the imported file ]


    3. { child file || [childPage/childPage.tsx] } => should contain following contents by order :
      1. Importing each child component file [that been used in the current sub main file] by it's path in the project files tree
      2. Define the [Type object] that used to define the main function [childPage()]'s props datatypes -> [ incase of defining props in the main function ]
      3. Define the [Sub main function]
      4. Assign the value of the passed props from the imported files
      5. Exporting the defined component function as a default function => [to able to read from the imported file ]

{ Arranging reactJs project } importances :

  • easier and more organized writing code
  • easier handling and reading

Conceptual practical examples about arranging of [mainPage.tsx] & [componentChild.tsx ] and reactJs project:

project's file tree , as following :

  1. [app/page.tsx] {Main file} => including all components files
  2. [Header()] { Sub main child file } => including the following child => [Header1()] & [has defined props]
  3. [Navbar()] { Sub main child file } => including the following child => [Item1()] & [has NO props]
  4. [Description()] {child file } => Not including any children components & [has defined props]
  5. [List()] {child file } => Not including any children componets & [has NO props]


  1. 1] {Main file || [app/page.tsx]} , with following steps =>
    - Importing the child defined component with in exteranl file page
    + Define the {type object} of the main component
    + Define the {main component function}
    + Exporting the {main component function} as default function

    // A] Importing each defined children components functions [that will be included in this main page directly ] from the other page :
    import Header from ' Header/Header.tsx' ;
    import Navbar from 'Navbar/Navbar.tsx' ;
    import Description from 'Description/Description.tsx' ;
    import List from 'List/List.tsx' ;

    // B] Define the main function of main component {Home() page } [with using best practise of using default at last of the script] :
    function Home() {
    // using return to include all page elements :
    return (
    < main className = {style.main} >
    // Including a [Sub main child] component function [with assigning value of the defined props ] :
    < Header {propName = Value} / >
    // Including a [Sub main child] component function [without passed props ] :
    < Navbar / >
    // Including a [child component] function [with assigning value of the defeined props ] :
    < Description / >
    // Including a [child component] function [ without passed props ] :
    < List {propName = value} / >
    // including a simple element in to the main page :
    < p > Hello world < /p >
    < /main >
    ) ;
    }

    // C] Exporting the main compoonent function at the last of script [best practise] :
    export default Home ;



  2. 2] {Sub Main Child file || [Header/Header.tsx]} =>
    - Importing he used child component functions
    + Define the {type object} of this child component
    + define {chlid component function}
    + Exoprting the defined the component function as the default fuction :

    // A] Importing each Used defined children components functions from the other child page [Header1.tsx] :
    import Header1 from 'Header1/Header1.tsx' ;

    // B] Define the type object to Used in defining children components functions's prop(s) :
    type headerParams = {
    headElem = string
    }

    // C] Define the [Sub main child] component function { Header() } :
    function Header(props : headerParams) {
    // using return to include this elements & included defined children components functions + with assinging the props datatype :
    return (
    // Define datatype of the declared prop [within the an Element] :
    < div > {props.headElem} < /div >
    // Including the imported child component :
    < Header1 / >
    ) ;
    }

    // Exporting the [Sub main compoonent function 'header'] to be used by the imported function file :
    export default Header ;

    // D] {Header1/Header1.tsx} => Define the included child component [Header1()] :
    function Header1() {
    // using return to add a simple element (without using brackets) :
    return < div > This is a child header1 < /div > ;
    }

    // Exporting the [child compoonent function 'header1'] to be used by the imported function file :
    export default Header1 ;





  3. 3] {Sub Main Child file || [Navbar/Navbar.tsx]} =>
    - Importing the used child component functions
    + Define {chlid component function}
    + Exoprting the defined the component funsction as the default fuction :

    // A] Importing each Used defined children components functions from the other child page [Item1.tsx] :
    import Item1 from 'Item1/Item1.tsx' ;

    // B] Define the type object to Used in defining children components functions's prop(s) :
    => this conponent not need a type object ,due it has no props
    // C] Define the [Sub main child] component function { Navbar() } :
    function Navbar() {
    // using return to include this elements & included defined children components functions :
    return (
    // Add a simple Element into the child :
    < div > this is a div < /div >
    // Including the imported child component [Item()] :
    < Item / >
    ) ;
    }

    // Exporting the [Sub main compoonent function 'header'] to be used by the imported function file :
    export default Navbar ;

    // D] {Item1/Item1.tsx} => Define the included child component [Item1()] :
    function Item1() {
    // using return to add a simple element (without using brackets) :
    return < div > This is a child Item1 < /div > ;
    }

    // Exporting the [child compoonent function 'Item1'] to be used by the imported function file :
    export default Item1 ;





  4. 4] {Child file || [Description/Description.tsx]} =>
    + Define the {type object} of this child component [due it has defined props]
    + Define {chlid component function}
    + Exoprting the defined the child component function as the default fuction

    // A] Define the [type object] to be Used in defining children components functions's prop(s)'s datatype :
    type DescpParams = {
    descElem = string
    }

    // B] Define the [Child] component function { Description() } :
    function Description(props : DescpParams) {
    // using return to include this elements with assinging the props datatype :
    return (
    // Define datatype of the declared prop [within the an Element] :
    < div > {props.descElem} < /div >
    ) ;
    }

    // C] Exporting the [child compoonent function 'Descripton'] to be used by the imported function file :
    export default Descripton ;






  5. 5] {Child file || [List/List.tsx]} =>
    + Define {child component function}
    + Exoprting the defined the child component function as the default fuction

    // *] No need for Importing :
    // *] No need for type object , dur to this function has no props :
    // A] Define the [Child] component function { List() } :
    function List() {
    // using return to include this elements with react fragment :
    return (
    < >
    < div > This a single div element < /div >
    < p > This a single paragraph element < /p >
    < / >

    ) ;
    }

    // B] Exporting the [child compoonent function 'List'] to be used by the imported function file :
    export default List ;






VIP notes about Importing and including the imported component files :

  • In order to Import component function from [child component file] -> To the [main component file] => must use the next syntax :
    import componentFucName from '@/containerFolder/componentFucName.tsx' ;
    => due the main file is ouside the children files , so you need to access the container folder first , before open the child component file

  • To import child component file -> inside another child component => must use hte next syntax :
    import componentFucName from './componentFucName.tsx' ;
    => due all children components files are in one container folder , which is by best pracise is ['components']
