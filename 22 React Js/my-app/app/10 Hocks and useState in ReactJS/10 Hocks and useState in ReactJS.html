10 Hocks and useState in ReactJS concept :

{ Hocks } concept in ReactJS :

  1. Group of built-in fuctions within the {ReactJs} , to acheive the auto reacting on the component fuction in the current pages , without need ot relao or Rerendering
  2. Common Group types built-in fuctions within the {ReactJs} => is Statmenent MAnagent , such as : { usestate() }


{ useState() } concept in ReactJS :

  1. one of the statements management within the {ReactJs}, which used to acheive an Auto reating on the [whole component function] , due the chenage occuring in cetatin defined variable value
  2. In the next lessons => we will study other Built-in fuction of the statements management , with more accurate reacting scope


{ useState() } importance in ReactJS :

  • It is essntial to make a defiend varilabe is affecting on the current component function contents
  • The change in the useState() defined variable will make the reactJS Re rendering the whole current function , including the defined variable's value


{ useState() } Syntax in ReactJS :
const [variableName , setVariableName] = useState < Variable Datatype > (initial value) ;


VIP Notes to use { useState() } in ReactJS :

  • We must use the both variable name and variable setter to be define the useState() built-in function
  • To use the useState define variable within another defined procesing funtion , we must use the variabloe Setter and the variable Name itself inide the () within the reactJS Syntax
  • To use the useState define variable within component function , we must use the variable Name itself inide the {} within the reactJS Syntax
  • {useState()} Variable definition in the reactJs => can be used in both main or child component
  • {useState()} Variable definition in the reactJs => must be before the function return() , to be able read inside


Main Steps about using [UseState] to define variables with in ReactJs application :

  1. Importings :
    • [ 'use client' ; ] => must be at the TOP of the [page.tsx] that using [useState] variables defintion
  2. Syntax :

    The default variable Syntax :

    • { const } & { [varName, setVarName] } & { useState(initial Value) } || { < datatype > of useState } => Must all be used to define variable in the reactJs, as following :
      // using only the datatype => to define the useState() datatype [In Directly by conclusion] :
      const [varName , setVarName] = useState(initial Value) ;
      // using only the initial value to define the useState() datatype [Directly by detection] :
      const [varName , setVarName] = useState<datatype > () ;


    Strong Typed Syntax variable :

    • { const } & { [varName, setVarName] } & { useState(initial Value) } & { < datatype > of useState } => Must all be used to define variable in the reactJs, as following :
      // using both of the Initial value & Datatype => to define the useState() datatype :
      const [varName , setVarName] = useState<datatype > (initial Value) ;

    { variable setter } => used to control the variable itself , by calling it


{ useState() } practical examples in ReactJS =>> [Creating an application of several single and accomulated calculations operations of (2) numbers] Example

  • [Creating an application of several single calculations operations of a number] Example :

    Define [useState] variable of number datatype, using (2) assinging datatypes methods[Directly || Indriectly] , as following :

    1. [Directly] assinging datatype method => to use the < datatype > of the useState varialble
    2. [Directly] assinging datatype method => to use the useState(datatype) of the useState varialble

    Define several functions of each calculation operation [to set the variable value by single arithmetic operation] , by using (2) methods , as following:

    1. [Single calculation operation] method => to process the useState variable itself by 1 [+-*/]
    2. [Accumulated calculation operation] method => to process the multiple useState varialbles by [+-*/]

    Define several functions of each calculation results [for accumulated calculations operations]

    Creating several buttons to activate each calculation operation [single & Accumulated]

    Appliation steps as following :
    // Import the [use client] to make reactJs deal with useState as client side , not as server side :
    'use client' ;
    // Define the main Home component function :
    functon Home() {
    // 1] Define the [useState] first & second variable Syntax with using both [datatype of the useState] & a [initial Variable value] {for strongly typed} :
    const [number1 , setNumber1] = useState<number> (0) ;

    const [number2 , setNumber2] = useState<number> (0) ;

    // 2] Defining the results variables [ of Accumulated arithmetic operations] with initial valaue of [0] :
    const [addRes , setAddRes] = useState<number> (0) ;
    const [subRes , setSubRes] = useState<number> (0) ;
    const [mulRes , setMulRes] = useState<number> (0) ;
    const [divRes , setDivRes] = useState<number> (0) ;


    // 3] Defining the main processes functions of results variables of [Single arithmetic operations] :
    // a] Single Addition operation :
    const addSingle = () => {
    setNumber1(number1 + 1) ;
    }

    // b] Single Substration operation :
    const subSingle = () => {
    setNumber1(number1 - 1) ;
    }

    // c] Single Multiply operation :
    const multSingle = () => {
    setNumber1(number1 * 1) ;
    }

    // d] Single Devision operation :
    const divSingle = () => {
    setNumber1(number1 / 1) ;
    }


    // 4] Defining the main processes functions of results variables of [Accumulated arithmetic operations] :
    a] Setting function of the [Addition operation] Result of both variables :
    const addition = () => {
    setAddRes(number1 + number2) ;
    }

    b] Setting function of the [Substraction operation] Result of both variables :
    const substraction = () => {
    setSubRes(number1 - number2) ;
    }

    c] Setting function of the [Multiply operation] Result of both variables :
    const multiply = () => {
    setMultRes(number1 * number2) ;
    }

    d] Setting function of the [Devision operation] Result of both variables :
    const devision = () => {
    setDevRes(number1 / number2) ;
    }


    // Returning the function result [after defining the useState variables and it's setter function ]:
    return (
    < main >
    // A] Container of Displaying the {Single Arithmeitcs Opertations} [Number1] & [Number2] variables values inside a Label Element + using group of Buttons to control operation Functions :
    < div >
    // 1] Displaying the [current live value of (Number1) & (Number2)] :
    < label > first number = {number1} < /label >

    < label > second number = {number2} < /label >

    // 2] Controling Buttons Group of Arithmetics operations to [number1] & [number2] :
    < button onclick={addSingle} >
    click to +1 on Number 1
    < /button >
    < button onclick={subSingle} >
    click to -1 on Number 1
    < /button >
    < button onclick={multSingle} >
    click to *1 on Number 1
    < /button >
    < button onclick={devSingle} >
    click to /1 on Number 1
    < /button >
    < /div >


    // B] Container of Displaying the {Acummulated Arithmeitcs Opertations} [addition] & [substraction] Results functions variables live values inside a Label Element + using group of Buttons to control operation Functions :
    < div >
    // 1] Displaying the [current live value of (Number1) & (Number2)] :
    < label > Addition Result = {addRes} < /label >
    < label > Substration Result = {subRes} < /label >
    < label > Multiply Result = {multRes} < /label >
    < label > Devision Result = {devRes} < /label >

    // 2] Controling Buttons Group of Arithmetics operations to [number1] & [number2] :
    < button onclick={addition} >
    click to Number1 + Number2
    < /button >
    < button onclick={substraction} >
    click to Number1 - Number2
    < /button >
    < button onclick={multiplay} >
    click to Number1 * Number2
    < /button >
    < button onclick={devision} >
    click to Number1 / Number2
    < /button >
    < /div >
    < /main >
    );

    }

    export default Home ;
