3 Initializing ReactJS App by VScode concept :

{package.json} file =>

  • This file include the main of any react app or page
  • It is including the main configuration files of the reactJs project , which are :
    1. {basic app information} section => which contains the basci and main information of this ReactJS app , such as :
      1. 'name' : the app name
      2. 'version' : the app version
      3. 'private' : the value of the privacy property => {ture: private|| false: public }
    2. {dependencies} section => which contains the versions and edition of the Frameworks , Libraries and packages used in this app
    3. { script} section => which contains the main commands orders and it's alises' to be used in this app , such as :
      • 'dev' => to run [NextDev]
      • 'build' => to run [next build] (to get the final resuilt by JS)
      • 'start' => to run your app on [server]
      • 'lint' => to scan your app from Errors and Bugs


{tsconfig.json} =>

  • contain all configurations compilation option of the Typescript , such as : ['target' : es5 || es6] which is the vesion of the ecma script used in this reactJs app


{app} folder =>

  • All files here are written by using Mergered languages (between JS & HTML & XML) || (between TS & HTML & XML) specialized for reactJs => called [TSX] or [JSX] , with foolwoing details :
    1. main deffereneces between {.tsx} & {.jsx}

      {.jsx} files => [Hybrid ergered languages (between JS & HTML & XML)]

      {.tsx} files => [Hybrid Mergered languages (between TS & HTML & XML)]

  • Include all main project files [one file for each page] (with extension: '.tsx' | '.jsx' ) , such as :
    1. {page.tsx} => This is the main reactJS conponent in the project root main page , which will render all other inner pages
    2. {page.tsx} => main parts or contents , as following :
      • {import} => which is the keyword of importing the Modules || packages || other inner pages contents components , whether it is in the same project folder or from outside
      • { export default of main function} => which is the main component HTML with default name [Home()]l , with follwonig properties :
        1. this function is rendering all home elements and componets
        2. this function is using {default} keyword to be as main funcion to be rendering (for reactJs and NextJS)
        3. the keyword {default} can be used at the end of the script, and not to be used at the main function definition
      • at general , there are (2) types function in {reactJS} :
        • { class component} => will be studied later
        • {functional component} => like the main component functon , which featured with these features :
          1. has curly brackets {}
          2. has {return} keyword [with using () after or not using ] , and written by using [.jsx] or [.tsx]
  • The {.jsx} and {.tsx} files is being written by these hybrid language , but after compilataion conveted to pure Javascript to be readable fort the broweser
  • The usages of {.jsx} or {.tsx} files extension in your react app is determined by {boiler plate} in the step of choosing [TS : .tsx] or [not : .jsx]


Run Terminal to execute the {Page.tsx} file , with following procedures :

  1. Starting the command line of the currrent opened app from => [Terminal -> new terminal -> cmd] and write the next command :
    { npx run dev } => [to Initializing {local host -> by port -> localhost:3000 } to (run/preview) the current (app/page) with ]
  2. The generated link should be clicked and choose the browser to open the (app/page) by it -> by using [ctrl + Lclick]
  3. This will open {live preview} of the current application => will be live respond to any changes happend in this page file script (once the script is saved)
  4. To acheive more dynamic respoding in live preview , you can activate the {auto save} feature from : [file -> auto save]
