E] Differences between library and framwork :

  • [Library] =>
    1. concept => A group of certain functions to perform particaual tasks
    2. Requirements => Neeed to be downloaded manually in you app files from exteral source , and need to certaim configurations to be ready for activation
    4. Examples => [ReactJS , JQuery]

  • [Framework] =>
    1. concept => a collection of Libraries to perform a certian methodology
    2. Requirements => Provided with builtin functios and configurations setteings , and ready to be used in your app using this framework
    4. Examples => [Angular , Vue , NextJs , Bootstrap]

  • [Reacts] => a Library of advanced Js , and need to install alot of functional components , and special configurations
  • [NextJs] => a framework made for ReactJs production , which is provided with builtin (which can be imported in to your project) functions of using ReactJs + with Advanced features


F] Analyze {Next.js} concept :

  • The main terminology of NextJS is [The React Framework for production] , which can be analyzed as following parts :
    1. [ReactJs] => We still writing using ReactJs [JSX] & [TSX] in our projects pages scripts , and using the new features of ReactJs such as : ,
      • [props]
      • [Hocks]
      • [Stats]
      • [Contex]

    2. [Framework] => We add more enhanced featues on the reactJS app such as [Routing] and more features , these new features providing a lot of benefites when using them to the project , as folowwing:
      • Solving common problems
      • Providing clear Guidance on new Features and how to use them
      • Saving time and Resources of installing exteranl libraries and set it's configurations

    3. [Production] => NextJS solving sever issues was facing the ReactJs , realted to , following aspects :
      1. Deploying app files
      2. Storing app files
      3. Executing app files from the server


G] NextJS Framework Features and Benefits :

  1. [File-Based Routing] :
    1. you dont need to install exteranl Library sources , such as [React Dom] for ReactJs routings , and use instead the built-in nextJS [file/folder Routing]
    2. Routing is very simple methodology , which allowing to create page router by simple calling its file/folder name allocated to this page to load the called functional component into the [SPA] single one page
    3. Routing in Nextjs is not need for special or complicated configurations settings to be used
    4. Routing methods in NextJs has (2) ways to be set in the reactJs application [app router] || [page file/folder] , which is being assinged with in the [boiliing plates in the {create-next-app}]

  2. [Server-Based Rendering] :
    1. A new concept in the Next to provide much easier method Auto optimized [SEO] , by using advanced Brower dev tool [light house] , which will be studied in details later, using new builtin concepts , such as :
      • {Pre-Rendering} : which is a new Tool in the nextjs helps in building your app according ot SEO
      • {Static side Generation} : which is a new Tool in the nextjs helps in building your app in static side according ot SEO
      • {Image optimization} : to be supported to the SEO

  3. [Fullstack compatablities] via [End Points] :
    1. By providing a new tool called [End Points] , which created to simplify the dealing with the sever and database , through the app
    2. By creating this tool inside the app => will provide an auto authentication for DATA [such as (signin) and (signout)] , without need to add custom backend p.language , such [python, Java]
    3. The [End Points] can handle the dealing with DataBase Schema and processed such as [store, edit , call]


    4. Requirements for coding :

    • Install [Node.js] environment runtime =>

      used to setup/Install and set main configurations of the project app files and folders , using [Boiling Plate] method

      including both [NPM] & [NPX] :

      1. [NPM] => the [Node package Manager] , which used to install and deal with the project's packages and libraries
      2. [NPX] => the [Node package Executer] , which used to execute the node code and commmand to implemented in the project

    • Install [vscode] + [Prettier Extension] => to used to write the project files scripts and debugin for bugs and errors
