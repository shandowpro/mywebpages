3 pages or app folder and creating Routers concept :

{Page folder} closer look =>

  1. Main contents and includes :

    This folder Includes the proejct main page and inner pages

    All pages in NextJS uses the ReactJs [JSX] & [TSX]

    Each page is represented by defining a functinoal component , whether is was a home page of a inner page

    Each page Must have a router to navigate to this page

    Home page path start with ['/']

    Inner pages path start with ['/pageRouterName'] {Regardless of used Router method}

    Any page Router Name must start with [Small letter]

    Any page functional component Name must start with [Capital letter]



  2. {_app.js} file :

    This file contains the constructor function used by [NextJs] to create any funcitonal component

    The Constructor component funciton is used within hte nextJs instead of using Extending Classes in old ReactJS

    The Constructor component funciton syntax as following :
    function MyApp({Component , PageProps}) {
    return < Component {...PageProps} / > [Returning the Conmponent function with estructuring all builtin properites NextJs ]
    }

    export default MyApp ;





Creating the {Home component} function within {SPA} single page application =>

  1. The home page is the [main page] {landing page} in the project
  2. The home page is an [index.js] File inside the main direct [page] folder
  3. It is about functional component , which is returning it's contents including imported components and { <main≶ } JSX element
  4. The syntax of [Home] functional component as following :
    function Home() {
    return (
    < > => [ start of {< >} fragements , which used when the functional component return several elemnts
    < ComponentName / > => [calling another exteranl defined component funtcion]

    < main > => [This is the main element that hold body element of the current functional component]
    < div >
    < h1 > This is Header < /h1 >
    < / div >
    < / main >
    < /> => [ end of {< >} fragements]
    ) ;
    }

    export default MyApp ; => [any functional component must be exported to be able read to the NextJS ]





Creating {Inner Pages} and routers =>

  1. The project can include several inner pages
  2. The Inner page is an [index.js] File created and existed one of following options acccording the used Router method :
    1. using [Folder Router method] => {index.js} file created inside a Sub folder in the main [page] folder -> [pages/innerPageName/index.js]
    2. using [File Router method] => {pageName.js} file created inside the main [page] folder direclty -> [pages/innerPageName.js]
  3. Inner page is about functional component also , which is returning it's contents including imported components and JSX element [without { <main≶ }]
  4. The syntax of this functional component as following :
    function InnerPage() {
    return (
    < > => [ start of {< >} fragements , which used when the functional component return several elemnts
    < ComponentName / > => [calling another exteranl defined component funtcion]

    < div >
    < h1 > This is Header < /h1 >
    < / div >
    < /> => [ end of {< >} fragements]
    ) ;
    }

    export default InnerPage ; => [any functional component must be exported to be able read to the NextJS]





{Router} in details [within NextJS] :

  1. {Router} Types [File Router] & [Folder Router] Methods [within NextJS] :
    • There are (2) methods to create a page Router [main page || inner page ] , to use it to navigate to this page in the server :
      1. {File Router} Method =>

        To create the page file by name of the functional Component [in small letter]

        The created page file must be inside the [page] folder directly


      2. {Folder Router} Method =>

        To create new [sub folder] by functional Component Name [in small letter]

        The created [sub folder] must be inside the main [page folder]

        The created [sub folder] must contain the page file with name of [index.js]



  2. [Nested Router] concept & Benefits :
    • It is about creating Router of a page and another page router inside the parent router
    • Nested Router Steps :

      Using the [Folder Router] Method to create the [parent Router] and it should include the folowing (2) files :

      1. {index.js} -> which represent the [parent Router page] (according to the folder Router )
      2. {nestedPage.js} -> which is the [child Router] (according to file method)



  3. {Router} Best practise [within NextJS] :
    1. In case of creating [Single Inner Page] => Recommended the next steps :

      use the {File method} in the main [Page folder]


    2. In case of creating [Nested inner page] => => Recommended the next steps :

      use the {Folder method} in the Sub Folder of parent

      use the {File method} of the child Router inside the Sub Folder of parent



  4. {Dynamic Router} concept & Benefits [within NextJS] :
    1. It is about a flexible page router by using one of built-in NextJS function [Router hocks] from 'next/router' [which is going to give a dynamic vlaue of the router within the server address path ]
    2. The common usage of the [dynamic router] => in online store with in related product inner pages depeneded on the product id value
    3. The [dynamic router] => must be with created by [file method] inside the nested parent Router folder
    4. The [dynamic router] => requirements needed as follwoing :
      1. Inside the {Dynamic Router} file Script :

        [inside the functional component and [[Before]] the (return) ] :

        • 1- Importing {useRouter} hock function form the [next/router] => as following :
          import {useRouter} from 'next/router' ;
        • 2- Define a variable to store the imported {useRouter} Hocks funciton :
          const routerVar = useRouter() ;
        • 3- Define a variable to store the Dynamic router from the qurery of built-in next Router hocks function by using [2 meothods] :
          // First method : by using [Normal variable] :
          const dynmRouterVar = routerVar.query.dynamicRuoterPage ;

          // Second method : by using [Destructuring variable] :
          const {dynmRouterVar} = routerVar.query ;


        [inside the functional component and [[inside]] the (return) ] :

        • using the defined Router varaible inside the return functional component :
          reutrn (
          < >
          < div >
          < h1 >
          Welcome to the page {dynmRouterVar}
          < h1 >
          < /div >
          </ >
          ) ;

    5. Important notes :
      1. Reageraless of the used method of defininig the Router varialbe => the defined dynamic variable must be typically as the same of inner dynamic page
      2. The created dynamic inner page's file name => must be between [], to be added to the nextJS {next/router} list
      3. Incase of defining the router varialbe with the normal variable we numst use the typlical spelling name of the created dynamic router file
      4. In case of using the desturucturing method of definein the router variable => we NOT use the dynamic router file in the definition
